LGOJ #P4151. 「WC2011」最大 XOR 和路径

题目描述

$\operatorname{XOR}$（异或）是一种二元逻辑运算，其运算结果当且仅当两个输入的布尔值不相等时才为真，否则为假。$\operatorname{XOR}$ 运算的真值表如下（$1$ 表示真，$0$ 表示假）：

而两个非负整数的 $\operatorname{XOR}$ 是指将它们表示成二进制数，再在对应的二进制位进行 $\operatorname{XOR}$ 运算。

故 $12 \operatorname{XOR} 9 = 5$。

容易验证， $\operatorname{XOR}$ 运算满足交换律与结合律，故计算若干个数的 $\operatorname{XOR}$ 时，不同的计算顺序不会对运算结果造成影响。从而，可以定义 $K$ 个非负整数 $A_1, A_2, \ldots, A_{K - 1}, A_K$ 的 $\operatorname{XOR}$ 和为 $A_1 \operatorname{XOR} A_2 \operatorname{XOR} \ldots \operatorname{XOR} A_{K - 1} \operatorname{XOR} A_K$。

考虑一个边权为非负整数的无向连通图，节点编号为 $1$ 到 $N$，试求出一条从 $1$ 号节点到 $N$ 号节点的路径，使得路径上经过的边的权值的 $\operatorname{XOR}$ 和最大。

路径可以重复经过某些点或边，当一条边在路径中出现了多次时，其权值在计算 $\operatorname{XOR}$ 和时也要被计算相应多的次数，具体见样例。

输入格式

输入文件 xor.in 的第一行包含两个整数 $N$ 和 $M$， 表示该无向图中点的数目与边的数目。

接下来 $M$ 行描述 $M$ 条边，每行三个整数 $S_i, T_i, D_i$，表示 $S_i$ 与 $T_i$ 之间存在一条权值为 $D_i$ 的无向边。

图中可能有重边或自环。

输出格式

输出文件 xor.out 仅包含一个整数，表示最大的 $\operatorname{XOR}$ 和（十进制结果）。

样例

样例输入

5 7
1 2 2
1 3 2
2 4 1
2 5 1
4 5 3
5 3 4
4 3 2

样例输出

6

样例解释

如图，路径 $1 \rightarrow 2 \rightarrow 4 \rightarrow 3 \rightarrow 5 \rightarrow 2 \rightarrow 4 \rightarrow 5$ 对应的 $\operatorname{XOR}$ 和为 $2 \operatorname{XOR} 1 \operatorname{XOR} 2 \operatorname{XOR} 4 \operatorname{XOR} 1 \operatorname{XOR} 1 \operatorname{XOR} 3 = 6$

当然，一条边数更少的路径 $1 \rightarrow 3 \rightarrow 5$ 对应的 $\operatorname{XOR}$ 和也是 $2 \operatorname{XOR} 4 = 6$。

数据范围与提示

对于 $20 \%$ 的数据，$N \leq 100, M \leq 1000, D_i \leq 10^{4}$；

对于 $50 \%$ 的数据，$N \leq 1000, M \leq 10000, D_i \leq 10^{18}$；

对于 $70 \%$ 的数据，$N \leq 5000, M \leq 50000, D_i \leq 10^{18}$；

对于 $100 \%$ 的数据，$N \leq 50000, M \leq 100000, D_i \leq 10^{18}$。

题解

线性基。

考虑随便找出一条从 $1 \rightarrow n$ 的路径。这样的路径自然异或和不一定最大，所以我们需要用某个环来改变它的异或值。

例如我们可以用环 $3 \rightarrow 6 \rightarrow 5 \rightarrow 3$ 来增广路径 $1 \rightarrow 4 \rightarrow 3 \rightarrow 5$，然后路径变为 $1 \rightarrow 4 \rightarrow 3 \rightarrow 6 \rightarrow 5 \rightarrow 3 \rightarrow 5$。

这里有个不太容易想到的地方：对于任意简单环，对于任何 $1 \rightarrow n$ 的路径，都可以直接增广，而不需考虑是否经过该环上的点。

原理其实很简单。考虑一个环 $P: P_1 \rightarrow P_2 \rightarrow \ldots \rightarrow P_k = P_1$。我们可以直接从 $1 \rightarrow P_1$，然后经过环 $P$，最后再按照原路 $P_1 \rightarrow 1$ 返回。这样一来，$1 \rightarrow P_1$ 路径上的边经过了两次，被抵消，只有环上的路径造成了影响。

用 DFS 找出所有必要的简单环，然后用线性基求出异或最大值即可。

时间复杂度：$O(n \log d_i)$。

代码

#include <algorithm>
#include <cstdio>

const int MAXLOG = 62;
const int MAXN = 100000 + 10;

typedef long long int64;

int n, m;

int64 base[MAXLOG + 1];

int64 dis[MAXN];
int pre[MAXN << 1], to[MAXN << 1], head[MAXN], wcnt;
int64 wdis[MAXN << 1];

inline void addedge(int u, int v, int64 w) {
    wcnt++;
    pre[wcnt] = head[u];
    head[u] = wcnt;
    to[wcnt] = v;
    wdis[wcnt] = w;
}
inline void add2edge(int u, int v, int64 w) {
    addedge(u, v, w);
    addedge(v, u, w);
}

inline void insert(int64 x) {
    int i;

    for (i = MAXLOG; ~i; i--) {
        if (x >> i & 1) {
            if (base[i])
                x ^= base[i];
            else {
                base[i] = x;
                return;
            }
        }
    }
}
inline int64 querymax(int64 x) {
    int i;

    for (i = MAXLOG; ~i; i--)
        x = std::max(x, x ^ base[i]);
    return x;
}

#define v to[i]
void dfs(int u, int64 res) {
    static bool vis[MAXN];
    int i;

    dis[u] = res;
    vis[u] = true;
    for (i = head[u]; i; i = pre[i])
        if (vis[v])
            insert(res ^ wdis[i] ^ dis[v]);
        else
            dfs(v, res ^ wdis[i]);
}
#undef v

int main() {
#ifndef ONLINE_JUDGE
    freopen("P4151.in", "r", stdin);
    freopen("P4151.out", "w", stdout);
#endif

    int i, u, v;
    int64 w;

    scanf("%d %d", &n, &m);
    for (i = 1; i <= m; i++) {
        scanf("%d %d %lld", &u, &v, &w);
        add2edge(u, v, w);
    }
    dfs(1, 0);
    printf("%lld\n", querymax(dis[n]));
    return 0;
}
