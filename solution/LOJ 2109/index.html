题目描述

说起来，毕业之后 B 君也就见过 R 君两面而已。

R 君有一个 $n \times m$ 的数组 $\{x_{i,j} \}$ ($1 \leq i \leq n, 1 \leq j \leq m$)。对于 $1 \leq i \leq n; 1 \leq j \leq m$，满足 $0 \leq x_{i, j} \leq m$。求可能的数组 $\{x_{i,j}\}$ 的解数。

B 君觉得限制太宽松，还要求对于 $1 \leq i \leq n; 1 \leq j<m$，满足 $x_{i,j} <x_{i,j+1}$，对于 $1 <i \leq n, 1 \leq j<m$，满足 $x_{i,j} <x_{i-1,j+1}$。

B 君认为 R 君可以直接 pwn 掉这个题。

R 君说：「黑的实在逼真 =.=，你起码把解数模 $1000000007$ 吧。」

B 君觉得 R 君说的有道理，于是想让你求解数模 $1000000007$ 的结果。

输入格式

一行两个整数表示 $n, m$，含义如题目中所述。

输出格式

一行一个数表示同时满足 B 君和 R 君的条件的 $\{x_{i,j}\}$ 的解数，模 $1000000007$ 的结果。

样例

样例输入

3 3

样例输出

40

数据范围与提示

对于 $100 \%$ 的数据，$1 \leq m, n \leq 10^6$。

题解

注意到 $0 \leq x_{i, 1} < x_{i, 2} < \ldots < x_{i, m} \leq m$，这一个有趣的限制告诉我们：我们一定可以把该序列表示为类似于 $0, 1, \ldots, k - 1, k + 1, k + 2, \ldots, m - 1, m$ 的形式。

我们用 $A_k$ 表示序列 $0, 1, \ldots, k - 1, k + 1, k + 2, \ldots, m - 1, m$，这里 $k \in [0, m]$。显然，$A_{k, i} = i - 1 \: (i \leq k), A_{k, i} = i \: (i > k)$。

对于序列 $A_i, A_j$，若 $A_j$ 可以接在 $A_i$ 的下面，就有 $A_{j, k} < A_{i, k + 1}$。可以证明，只有当 $j + 1 \geq i$ 时，$A_j$ 可以接在 $A_i$ 下面。

那么就可以 DP 了。设 $f[i][j]$ 表示前 $i$ 行组成的序列，第 $i$ 行的序列为 $A_j$，那么可得 $f[i][j] = \sum_{k = 0}^{j + 1} f[i - 1][k] = f[i][j - 1] + f[i - 1][j + 1]$。边界条件是 $f[1][0 \ldots m] = 1$。

直接 DP 是 $O(n^2)$ 的。让我们考虑怎么优化。

我们注意到，我们可以把它看成一个棋盘上的游走问题。也就是说，从 $(1, 1)$ 开始，每次可以向右或向左下方向走，求到达 $(n, m)$ 的方案数。

o --- o --- o
     /     /
    /     / 
   /     /
  /     /
o --- o --- o
     /     /
    /     / 
   /     /
  /     /
o --- o --- o

斜着走太奇怪了。我们可以把第 $i$ 行向右移动 $(i - 1)$ 个单位长度，就把斜着走变成了竖着走。

o --- o --- o
      |     |
      |     |
      |     |
      |     |
      o --- o --- o
            |     |
            |     |
            |     |
            |     |
            o --- o --- o

为了方便，我们上下翻转，得

            o --- o --- o
            |     |
            |     |
            |     |
            |     |
      o --- o --- o
      |     |
      |     |
      |     |
      |     |
o --- o --- o

可以把它放在坐标系下考虑。建立点 $S(0, 0), T(n + m + 1, n)$，直线 $y_1 = x + 1, y_2 = x - m - 2$，那么问题就转化为了：从 $S$ 出发，每次只能向右或向上走，只能在格点上，求在不碰到 $y_1, y_2$ 的前提下到达 $T$ 的方案数。

根据容斥原理，总方案数等于「没有限制走到点 $T$ 的方案数」-「只穿过 $y_1$ 走到点 $T$ 的方案数」-「只穿过 $y_2$ 走到点 $T$ 的方案数」-「穿过 $y_1$ 和 $y_2$ 走到点 $T$ 的方案数」。

但这样太复杂了，后面那个东西基本算不出来。换个方法。

根据容斥原理，总方案数等于「没有限制走到点 $T$ 的方案数」-「第一次穿过 $y_1$ 走到点 $T$ 的方案数」-「第一次穿过 $y_2$ 走到点 $T$ 的方案数」。

没有限制走到点 $(x, y)$ 的方案数为 $\mathrm{C}^x_{x + y}$。因此，没有限制走到点 $T$ 的方案数为 $\mathrm{C}^n_{2n + m + 1}$。

第一次穿过 $y_1$ 的方案数，我们可以将 $T$ 对于 $y_1$ 对称得到点 $T'$，则方案数等于「没有限制到达 $T'$ 的方案数」-「第一次穿过 $y_2$ 到达 $T'$ 的方案数」。$y_2$ 同理。

可以发现，这两个问题是递归的。

设 $f(x, y)$ 表示第一次穿过 $y_1$ 到达 $y_1$ 上方的点 $(x, y)$ 的方案数，$g(x, y)$ 表示第一次穿过 $y_2$ 到达 $y_2$ 下方的点 $(x, y)$ 的方案数，那么有

$$ \begin{aligned} f(x, y) &= \mathrm{C}^x_{x + y} - g(y + m + 2, x - m - 2) \\ g(x, y) &= \mathrm{C}^x_{x + y} - f(y - 1, x + 1) \\ \end{aligned} $$

边界是 $f(x, y) = g(x, y) = 0 \: (\min\{x, y\} < 0)$。

可以证明，这样的复杂度不会超过 $O(n)$。预处理阶乘后递归计算即可。

时间复杂度：$O(n)$。

代码

#include <algorithm>
#include <cstdio>

typedef long long int64;

const int MAXN = 4000000 + 10;
const int MOD = 1000000007;

int fac[MAXN], invfac[MAXN];
int n, m;

inline int qmod(int x) { return (x >= MOD) ? (x - MOD) : x; }
inline int qpow(int64 b, int p, int mod) {
    int res = 1;

    while (p) {
        if (p & 1)
            res = res * b % mod;
        b = b * b % mod;
        p >>= 1;
    }
    return res;
}
inline int C(int n, int m) { return 1ll * fac[m] * invfac[n] % MOD * invfac[m - n] % MOD; }
inline int f(int, int);
inline int g(int, int);
inline int f(int x, int y) {
    if (x < 0 or y < 0)
        return 0;
    else
        return qmod(C(x, x + y) - g(y + m + 2, x - m - 2) + MOD);
}
inline int g(int x, int y) {
    if (x < 0 or y < 0)
        return 0;
    else
        return qmod(C(x, x + y) - f(y - 1, x + 1) + MOD);
}

int main() {
#ifndef ONLINE_JUDGE
    freopen("2109.in", "r", stdin);
    freopen("2109.out", "w", stdout);
#endif

    int i, ans;
    int tx, ty;

    scanf("%d %d", &n, &m);
    fac[0] = invfac[0] = 1;
    for (i = 1; i < MAXN; i++)
        fac[i] = 1ll * fac[i - 1] * i % MOD;
    invfac[MAXN - 1] = qpow(fac[MAXN - 1], MOD - 2, MOD);
    for (i = MAXN - 2; i; i--)
        invfac[i] = invfac[i + 1] * (i + 1ll) % MOD;

    tx = n + m + 1;
    ty = n;
    ans = C(tx, tx + ty);
    ans = qmod(ans - f(ty - 1, tx + 1) + MOD);
    ans = qmod(ans - g(ty + m + 2, tx - m - 2) + MOD);
    printf("%d\n", ans);
    return 0;
}
