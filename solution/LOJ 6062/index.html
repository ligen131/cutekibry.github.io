题目描述

给出一个长度为 $n$ 的数列 $\{a_i \}$ 和一个长度为 $m$ 的数列 $\{ b_i \}$，求 $\{ a_i \}$ 有多少个长度为 $m$ 的连续子数列能与 $\{ b_i \}$ 匹配。

两个数列可以匹配，当且仅当存在一种方案，使两个数列中的数可以两两配对，两个数可以配对当且仅当它们的和不小于 $h$。

输入格式

第一行三个数字 $n, m, h$。

第二行有 $m$ 个数字 $b_1, b_2, \ldots, b_m$。

第三行有 $n$ 个数字 $a_1, a_2, \ldots, a_n$。

输出格式

输出一个数字，$\{a_i\}$ 有多少个长度为 $m$ 的连续子数列能与 $\{b_i\}$ 匹配。

样例

样例输入 1

5 2 10
5 3
1 8 5 5 7

样例输出 1

2

样例输入 2

2 2 6
2 3
3 4

样例输出 2

1

样例输入 3

4 2 10
5 5
9 3 8 9

样例输出 3

1

数据范围与提示

对于 $10\%$ 的数据，$1 \leq m \leq n \leq 10$；

对于 $40\%$ 的数据，$1 \leq m \leq n \leq 1000$；

对于 $100\%$ 的数据，$1 \leq m \leq n \leq 150000$；

对于 $100\%$ 的数据，$1 \leq a_i, b_i, h \leq 10 ^ 9$。

题解

容易发现，$b$ 的顺序是无关的。我们可以先对 $b$ 进行排序，这样连边就可以直接向 $[p, m]$ 连边了。

很容易想到一个贪心的思路。若 $a_i$ 可以向 $b_j \: (j \in [p_i, m])$ 连边，那么我们会优先向 $p_i$ 连边，这样可以给其他限制较多的 $a_i$ 腾出更多的空间。

另一个角度上说，我们可以把子序列的 $p_i$ 进行排序，那么当且仅当 $p_i \leq i$ 时该图可以匹配。

证明是很显然的。如果 $p_i > i$，说明要用前 $i$ 个数填满 $[1, p_i]$ 是不可能的。这也可以通过霍尔定理来证明。

用权值线段树维护。初始化序列为 $[-1, -2, -3, \ldots, -m]$，然后对于添加 $a_i$ 的情况令区间 $[p_i, m]$ 加 $1$，查看全局最小值是否为负数即可。

代码

#include <algorithm>
#include <cstdio>
#include <cstring>

#define lp (p << 1)
#define rp (p << 1 | 1)
#define mid ((l + r) >> 1)
#define lson lp, l, mid
#define rson rp, mid + 1, r

const int MAXN = 150000 + 10;
const int P = 1 << 18;

int a[MAXN], p[MAXN];
int b[MAXN];
int n, m, h;
int mark[P << 1];
int minv[P << 1];

inline void addnode(int p, int x) {
    mark[p] += x;
    minv[p] += x;
}
inline void down(int p) {
    if (mark[p]) {
        addnode(lp, mark[p]);
        addnode(rp, mark[p]);
        mark[p] = 0;
    }
}
inline void up(int p) { minv[p] = std::min(minv[lp], minv[rp]); }
void add(int p, int l, int r, int a, int b, int x) {
    if (a <= l and r <= b) {
        addnode(p, x);
        return;
    }
    down(p);
    if (a <= mid)
        add(lson, a, b, x);
    if (b > mid)
        add(rson, a, b, x);
    up(p);
}

int main() {
#ifndef ONLINE_JUDGE
    freopen("6062.in", "r", stdin);
    freopen("6062.out", "w", stdout);
#endif

    int i, ans;

    memset(minv, 0x3f, sizeof(minv));

    scanf("%d %d %d", &n, &m, &h);
    for (i = 1; i <= m; i++)
        scanf("%d", &b[i]);
    for (i = 1; i <= n; i++)
        scanf("%d", &a[i]);

    for (i = 1; i <= m; i++)
        minv[i + P] = -i;
    for (i = P; i; i--)
        up(i);

    std::sort(b + 1, b + 1 + m);
    for (i = 1; i <= n; i++)
        p[i] = std::lower_bound(b + 1, b + 1 + m, h - a[i]) - b;

    for (i = 1; i <= m; i++)
        add(1, 0, P - 1, p[i], m, 1);
    ans = minv[1] >= 0;
    for (i = m + 1; i <= n; i++) {
        add(1, 0, P - 1, p[i - m], m, -1);
        add(1, 0, P - 1, p[i], m, 1);
        ans += minv[1] >= 0;
    }
    printf("%d\n", ans);
    return 0;
}
