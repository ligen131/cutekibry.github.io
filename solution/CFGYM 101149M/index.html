题目描述

这是一道交互题。

有一个长度为 $n$ 的排列 $A_1, A_2, \ldots, A_n$，元素的值是未知的。

你的程序可以输出 ? x y，表示询问元素 $A_i, A_j$ 的大小关系。如果 $A_i < A_j$，交互器会输出 <，否则输出 >。

请在 $(n + 24)$ 次询问内求出序列中次大值的下标，并输出 ! x，其中 $x$ 为序列次大值的下标。

输入格式

第一行，一个整数 $n \: (2 \leq n \leq 1000)$，表示排列长度。

输出格式

多行，表示你的程序的询问。

最后一行，字符串 ! x，其中 $x$ 为序列次大值的下标。

样例

样例输入

5
>
>
>
>
>
<
>

样例输出

? 1 2
? 1 3
? 1 4
? 1 5
? 2 3
? 2 4
? 4 5
! 4

样例解释

可能的排列为 $[5, 4, 3, 2, 1]$ 或 $[5, 4, 1, 3, 2]$。

题解

Blogewoosh #6 评论区 dalex 提到的问题，算是该随机算法的模板题了。

维护目前最大值、次小值的下标 $m1, m2$。

首先询问 $A_i$ 是否大于次大值，这需要 $1$ 次询问。

如果 $A_i$ 大于次大值，再询问 $A_i$ 是否大于最大值，根据结果更改 $m1, m2$，这需要 $1$ 次询问。

怎么证明期望询问约 $n + \log n$ 次呢？

显然，只有当 $A_i$ 大于次大值时，才会多出一次询问；此时次大值一定会变得更大。因为在打乱询问顺序后，$A_i$ 期望在 $[1, n]$ 间均匀随机，所以次大值变化的次数约为 $\log n$。

期望询问次数：$n + \log n$。

代码

#include <algorithm>
#include <cstdio>

typedef unsigned uint;

const int MAXN = 1000 + 10;

int n;
int pos[MAXN];
uint seed;

inline uint randint(uint l, uint r) { return (seed = seed * 2333ll + 10007) % (r - l + 1) + l; }
inline void shuffle(int a[], int l, int r, int t) {
    while (t) {
        std::swap(a[randint(l, r)], a[randint(l, r)]);
        t--;
    }
}

inline char query(int x, int y) {
    char ch;
    printf("? %d %d\n", x, y);
    fflush(stdout);
    scanf(" %c", &ch);
    return ch;
}
inline void submit(int x) {
    printf("! %d\n", x);
    fflush(stdout);
}

int main() {
    int i;
    int m1, m2;
    char x;

    scanf("%d", &n);
    for (i = 1; i <= n; i++)
        pos[i] = i;
    shuffle(pos, 1, n, n);

    m1 = pos[1];
    m2 = pos[2];
    if (query(m1, m2) == '<')
        std::swap(m1, m2);

    for (i = 3; i <= n; i++) {
        if (query(m2, pos[i]) == '<') {
            if (query(m1, pos[i]) == '<') {
                m2 = m1;
                m1 = pos[i];
            }
            else
                m2 = pos[i];
        }
    }
    submit(m2);
    return 0;
}
